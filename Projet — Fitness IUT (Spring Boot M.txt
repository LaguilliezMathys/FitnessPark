Projet — Fitness IUT (Spring Boot MVC + REST)
1) Contexte
Vous devez développer une application Spring Boot permettant de gérer des routines de fitness.
Une routine contient plusieurs exercices.
Routines : nom, description, date de création, statut (active/inactive)
Exercices : nom, répétitions, poids
Routines
id: Long «PK»
name: String
description: String
creation_date: LocalDate
status: String (active/inactive)
Exercises
id: Long «PK»
name: String
repetitions: Integer
weight: Double
routine_id: Long «FK»
contains
1 *

5) Validation (obligatoire)
Vous devez utiliser Bean Validation (@Valid + BindingResult) et afficher les erreurs dans les formulaires.
Routine
name : obligatoire, taille min/max (ex : 3 à 120)
status : obligatoire (active / inactive)
Exercice
name : obligatoire (taille min/max raisonnable)
repetitions : entier >= 1
weight : nombre >= 0
6) API REST (obligatoire)
L’API REST doit être accessible sous le préfixe /api.
Endpoints minimaux attendus :
GET /api/routines : liste des routines
GET /api/routines/{id} : détail d’une routine
POST /api/routines : création d’une routine
DELETE /api/routines/{id} : suppression d’une routine
Option recommandé (non obligatoire, mais valorisé) :
pagination / filtre sur GET /api/routines (paramètres page / size et/ou mc)
Remarque : vous pouvez renvoyer les exercices dans le détail, ou proposer un endpoint séparé (au
choix). L’important est que l’API soit utilisable et démontrée dans la documentation.
